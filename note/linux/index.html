# 一 Linux系统的目录结构

/ 根目录
    /boot: 启动目录，启动信息
    /bin: 程序的启动文件，程序的命令
    /sbin: 超级用户存放一些应用程序的地方
    /dev: 设备目录
    /etc: Editable Text Configuration，可编辑文本配置，放一些配置信息
    /home: 用户家目录
    /root: root用户单独的家目录
    /lib: 存放一些库
    /lib64: 给64为操作系统提供的，存放一些库
    /lost+found: 这个目录平时是空的，系统非正常关机留下“无家可归”的文件就在这里
    /media: 自动识别一些设备的时候，会挂载到这个地方，必须：CD/DVD
    /mnt: 安装临时文件系统的安装点，让用户临时挂载其他的文件系统
    /proc: 虚拟文件系统目录，是系统内存的映射。用来获取系统信息
    /tmp: 用于存放各种临时文件，是公用的临时文件存储点
    /urs: Unix System Resource, Unix系统资源，这个目录很重要，用来放一些应用程序
    /opt: 这里主要放一些可选的程序，譬如安装一些冉家的最新beta版本，用完直接删除，软件安装在此目录，一个软件只有一个目录，可以一次性完全删除，而/usr/local则不能
    /sys: 是sysfs文件系统的挂载点，sysfs是一种虚拟的基于内存的文件系统，它的作用除了与proc相同的具有查看和设定内核参数功能外，统一设备模型
    /selinux: Security-Enhanced Linux的缩写，用来保证系统的安全
        getenforce: 获取状态
        setenforce: 设置状态
        永久关闭:
            打开: /etc/selinux/config
            修改: /SELINUX-disabled
    /srv: 系统启动服务时可以访问的数据库目录
    /var: 用于存放运行时需要改变数据的文件，比如各种服务的日志文件

# 二 网络及相关设置

  1. ping
    • 作用: 通常用于检测网络设备的连通性
    • 格式: ping ip/域名
    • 选项: -c，指定方式测试数据包的次数
  2. ifconfig
    • 作用: 查看或设置网卡信息
    • 实例:
      ifconfig: 查看活动网卡信息
ifconfig eth0 down: 关闭网卡，等价于ifdown eth0
ifconfig eth0 up: 开启网卡，等价于ifup eth0
    • 服务:
      /etc/init.d/network start | stop | restart  开启 | 关闭 | 重启 网络服务
service network start | stop | restart 开启 | 关闭 | 重启 网络服务
  • 网卡配置文件: /etc/sysconfig/network-scripts/ifcfg-eth0
    TYPE=Ethernet
PROXY_METHOD=none
BROWSER_ONLY=no
BOOTPROTO=dhcp               # 动态分批ip地址
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_FAILURE_FATAL=no
IPV6_ADDR_GEN_MODE=stable-privacy
NAME=enp0s3
UUID=eee027e6-e6ce-4aa2-a661-263471392ccb
DEVICE=enp0s3
ONBOOT=yes
  • 添加域名服务器(DNS)地址: /etc/resolv.conf
    • DNS1 = 8.8.8.8 #域名解析服务 google dns服务器
    • DNS2 = 114.114.114.114
  • 添加本地域名解析服务: /etc/hosts
    • 127.0.0.1 www.hello.com

# 三 服务及进程

  1. vmstat:
    • 结果:
      procs -----------memory---------- ---swap-- -----io---- -system-- ------cpu-----
    r  b   swpd   free   buff  cache   si   so    bi    bo   in   cs us sy id wa st
    2  0      0 1578008   2108 160164    0    0    87    14  218  281  3  6 92  0  0
    • 说明:
      • procs: 进程
        r 表示运行队列(就是说多数个进程真的分配到CPU)
b 表示阻塞的进程数
      • memory: 内存
        swpd 虚拟内存已使用的大小，如果大于0，表示你的机器物理内存不足了
free 空闲的物理内存的大小
buff 用来存储，目录里面有什么内容，权限等的缓存
cache 直接用来记忆我们打开的文件，给文件做缓存
      • swap: 交换
        si 每秒从磁盘读入虚拟内存的大小
so 每秒虚拟内存写入磁盘的大小
      • io: 块设备
        bi 每块设备每秒接收的块数量
bo 块设备每秒发送的块数量
      • system: 系统
        in 每秒CPU的中断次数，包括时间中断
cs 每秒上下文切换次数
      • cpu: 中央处理器
        us 用户CPU时间
sy 系统CPU时间
id 空闲CPU时间，一般来说，id+us+sy = 100
wa 等待IO的CPU时间
st 虚拟机占用的时间百分百，一般不用关心
  2. w
    • 作用: what的简写，当前系统在干什么
    • 结果:
      20:51:23 up 43 min,  3 users,  load average: 0.16, 0.22, 0.20
USER     TTY      FROM             LOGIN@   IDLE   JCPU   PCPU WHAT
kevin    tty1                      09:08   22:43   0.02s  0.02s -bash
root     pts/0    192.168.10.1     09:09    3.00s  0.04s  0.00s w
root     pts/1    192.168.10.1     09:09   41:53  10.34s 10.34s top
    • 说明:
      第一行: 系统当前时间  开机持续时间，登录用户个数，平均负载(分钟数)
其他行: 用户 | 终端 | 来源 | 登录时间 | 空闲时间 | 使用时间 | 当前进程时间 | 正在做
  3. top
    • 使用:
      w 的详细显示，每3秒刷新一次
shift + m可以按照所占内存排序
q 退出监测
    • 结果:

其他行 
    PID     进程号

  4. free
    • 作用: 查看内存
    • 选项: -h人性化显示
  5. ps
    • 作用: 查看进程
    • 选项:
      a 显示控制终端的所有进程
u 显示用户信息
x 显示没有控制终端的进程
    • 实例: px aux
  6. kill
    • 作用: 杀死一个进程
    • 选项: -9, 强制结束
    • 实例: kill -9 PID

# 四 YUM及RPM安装

  1. yum安装软件
    • 基本说明
      • yum相当于windows上面的360的软件中心，appstore，安卓的应用商店
      • yum是redhat系列发行版的软件安装命令，debian系列用的是apt-get
      • yum安装软件的来源需要存在一个地方，这个地方叫软件仓库，又叫yum源
      • yum源的配置文件在/etc/yum.repos.d/这个目录下面
    • 更换yum源
      • 进入目录 cd /etc/yum.repos.d
      • 备份文件 mv Centos-Base.repo CentOS-Base.repo.bak
      • 下载文件 curl -O http://mirros.163.com/.help/CentOS6-Base-163.repo
      • 修改名称 mv CentOS6-Base-163.repo CentOS-Base.repo
      • 清除缓存 yum clean all
      • 生成缓存 yum makecache
    • 基本格式 yum <操作> [选项]
    • 常用操作
      清空所有缓存        clean all
重新生成包信息缓存  makecache
安装指定软件        install
安装一组软件        groupinstall
更新指定软件        update
卸载指定软件        remove
搜索指定软件        search
显示所有软件        list
    • 常用选项
      默认确定操作    -y
只下载不安装    --downloadonly
指定下载目录    --downloaddir
  2. rpm安装软件
    • 说明:
      • rpm以前是Redhat Package Manager的缩写,现在是RPM Package Manager的递归缩写
      • 使用此方式安装的软件大多有依赖关系问题，通常一个软件需要依赖几个包
    • 格式: rpm [选项] 包名
    • 实例: 安装vim
      • 下载vim
        yum install vim -y --downloadonly --downloaddir=.
      • 安装vim
        rpm -ivh vim-filesystem-7.4.629-5.el6.x86_64.rpm
rpm -ivh vim-common-7.4.629-5.el6.x86_64.rpm
rpm -ivh vim-enhanced-7.4.629-5.el6.x86_64.rpm
  3. 源码安装软件
    • 配置: configure
    • 编译: make
    • 安装: make install
    • 实例演示，源码安装Apache

      • 下载C编译器，用来编译C语言编写的源代码

        安装gcc:        yum install -y gcc
查看gcc版本:    gcc -v

      • 下载Apache源码

        curl -O http://mirrors.tuna.tsinghua.edu.cn/apache//httpd/httpd-2.4.39.tar.gz

      • 解压下载的压缩包

        tar -zxvf httpd-2.4.39.tar.gz

      • 进入解压后的文件夹

        cd httpd-2.4.39

      • 编译前的配置: ./configure --prefix=/usr/local/apache2 --with-apr=/usr/local/apr/bin/apr-1-config --with-apr-util=/usr/local/apr-util/bin/apu-1-config

        • 配置选项
          • -h/--help 查看帮助
          • --prefix 指定安装目录

      • 编译源代码 make

        • 错误一
          collect2: error: ld returned 1 exit status
make[2]: *** [htpasswd] Error 1
make[2]: Leaving directory `/root/httpd-2.4.39/support'
make[1]: *** [all-recursive] Error 1

        • 编译失败后，再次编译之前需要运行
          • make clean 清除上次make命令所产生的object文件（后缀为“.o”的文件）及可执行文件
          • ldconfig 该命令通常在系统启动时运行，确保动态链接库为系统所共享。当用户安装了一个新的动态链接库时，则需手工运行该命令

      • 安装 make install

      • 控制Apache软件

        • 控制命令 PREFIX/bin/apachectl

        • 命令的位置/usr/local/apache2/bin/apachectl

        • 控制操作 start | stop | restart 启动 | 关闭 | 重启

        • 启动apache错误

          AH00558: httpd: Could not reliably determine the server's fully qualified domain name, using fe80::a475:8313:82cf:dd03%enp0s3. Set the 'ServerName' directive globally to suppress this message

          处理方法

          打开apache配置文件，开启并更改配置项ServerName

# 五 用户及用户组

  1. whoami
    • 说明: 查看当前登录的用户
    • 格式: whoami
  2. /etc/passwd
    • 说明: 该文件存放了系统中所有的用户，每一行的每一列如下
      用户名:密码:描述:用户ID/UID:组ID/GID:用户家目录:shell
  3. useradd
    • 说明: 添加(创建)用户
    • 格式: useradd 用户名
    • 选项:
      • -d 指定用户家目录
      • -u 指定用户id，必须大于500(小于500都已经被系统占用)
      • -s 指定用户执行的shell，若用户已经创建可以使用一些的方式禁止登陆 chsh 用户名 -s /sbin/nologin
  4. userdel
    • 作用: 删除指定用户
    • 格式: userdel 用户名
    • 说明: 彻底删除用户还需删除'/home/用户名'目录和'/var/mail/用户名'目录
  5. passwd
    • 作用: 设置用户密码
    • 格式: passwd [用户名]
    • 说明:
      • 修改指定用户的密码，若不指定则修改当前登录的用户密码
      • 查看所有用户的密码/etc/shadow
  6. su
    • 作用: 切换用户
    • 格式: su [用户]
    • 说明: 切换到指定用户，若不指定则切换到root用户
  7. sudo
    • 说明:
      • 若只想简单的只想一个超级用户的命令，不想切换用户
      • 格式是在原来的命令前面加上sudo即可
  8. 特殊标识
    • # 超级用户
    • $ 普通用户
    • ~ 用户家目录
  9. /etc/group 说明: 查看系统中所有的组
  10. groupadd
    • 作用: 添加用户组
    • 格式: groupadd 组名
  11. groupdel
    • 作用: 删除用户组
    • 格式: groupdel 组名
  12. gpasswd
    • 作用: 向用户组添加或删除用户
    • 格式: gpasswd [选项] 用户名 组名
    • 选项:
      • -a 将指定的用户加入指定组
      • -d 将指定用户从指定组删除
  13. chgrp
    • 作用: 设置文件所属的组
    • 格式: chgrp 组名 文件名
    • 说明: 修改文件所属的组
  14. chown
    • 作用: 设置文件的所有者
    • 格式: chown 用户[:组名] 文件名
    • 说明:
      • 修改文件的所有者[及组名]
      • 修改用户及组的时候，可以使用UID或GID
      • 加上-R参数可以递归修改子目录的用户及组

# 六 Linux文件与目录管理

# 6.1 目录与路径

. 此层目录

# 6.2 文件与目录管理

cp 复制文件或目录( copy )

rm 移动文件或目录( remove files or directories )

mv 移动文件与目录，或更名( move or rename files )

basename 取得最后的文件名( strip directory and suffix from filenames )

dirname 取得目录名( strip last component from file name )

# 6.3 文件内容查阅

  1. 文件内容查阅常用命令的英文缩写

    cat 由第一行开始显示文件内容( concatenate files and print on the standard output )

tac 由最后一行开始显示文件内容( concatenate and print files in reverse )

nl 显示的时候输出行号( number lines of files )

more 一页一页的显示文件内容( file perusal filter for crt viewing )

less 与more类似，可以向前翻页( opposite of more )

head 只看头几行( output the first part of files [ 默认前10行 ] )

tail 只看后几行( output the last part of files )

od 以二进制方式读取文件( dump files in cotal and other formats )

touch 修改文件时间或创建文件( change file timestamps )

  2. 文件内容查看的使用说明

    cat: 一次性查看文件所有内容
        cat filename
tac: 一次性倒叙查看文件所有内容
	tac filename
head: 
	作用: 显示文件开头的若干行内容
	格式: tail -n file n为要显示的行数
tail:
	作用: 显示文件结尾的若干行内容
	格式: tail -n file n为要显示的行数
more: 
    作用: 分页显示其它命令执行的结果
    格式: 其他命令  | more, 如：`cat file | more`
    说明:
        - 当内容显示一屏时停止
        - 空格向下翻页(只能向下)
        - 回车键向下显示一行
        - q键退出(结束查看)
less:
    作用: 分页显示其他命令执行结果
    格式: 其他命令 | less,如: `cat file | less`
    说明: 
        与more命令功能相同，多了上下按键上下翻一行
        more和less的前面可以是很多查询搜索等命令，如: ls、find

  3. 文件整体操作

    touch:
    作用: 创建普通文件
    格式: touch file1 [file2]
cp:
    作用: 拷贝文件
    格式: cp 源文件 目标文件
rm: 
    作用: 删除文件
    格式: rm 文件
mv: 
    作用: 移动文件
    格式: mv 源文件 目标文件
mkdir:
    作用: 创建目录文件
    格式: mkdir dir1 [dir2]
    说明:
        删除文件时会有提示，若不想提示可以添加参数'-f'
        删除或拷贝目录时需要添加参数'-r'
ln:
    作用: 创建链接文件
    格式: ln [-s] 源文件 目标文件
    硬链接: 不加'-s'选项时，简单理解为一个文件有多个名字
        1. 不占用实际空间
        2. 不允许给目录创建
        3. 只能跨文件系统
    软链接: (也叫特征链接)，添加'-s'选项时，简单理解为一个文件的内容是另一个文件的路径
        1. 类似于windows的快捷方式
        2. 可以对目录创建
        3. 可以跨文件系统

# 6.4 文件权限管理

  1. chmod

    • 作用: 修改文件的权限
    • 说明: ls -l中的第一列就是权限相关的信息，下面对其中的11个字符j进行说明
      • 1 文件类型(b/c/d/l/s/p/-)
      • 2/3/4 文件所有者的读(r)/写(r)/执行(x)权限，-表示不具备该权限
      • 5/6/7 文件所有者所属组的读写执行权限
      • 8/9/10 其他用户的读写执行权限
      • 11 .表示启用了selinux，空表示没有开启selinux
    • 参数:
      • r: 可读
      • w: 可写
      • x: 可执行
      • +: 添加权限
      • -: 去掉权限
      • =: 设置权限
      • u: 用户
      • g: 组
      • o: 其他
    • 数值表示: Linux中的权限以一组数据来表示，所有者/组/其他的权限分别以3位二进制表示，然后再将起写成8进制OK了，如: 755=> 111 101 101，结果就是: 所有者拥有读/写/执行权限，同组拥有读/执行权限，其他拥有读/执行权限
    • 实例:
      chmod +x file   给文件添加可执行权限
chmod -w file   给文件去掉可写权限
chmod u+x file  给文件的所有者添加可执行权限
chmod g-w file  给文件的同组去掉可写权限
chmod u=rwx,g=rx,o=rx file  所有者读写执行，同组读写，其他读写
chmod 0755 file 功能同上

  2. umask 文件默认权限( display or set file mode mask )

    • 作用: 查看或设置umask的值，umask用来确定创建文件的默认权限
    • 格式: umask [value]
    • 说明:
      • 若后面没有跟值表示查看默认权限，后面有跟值表示设置
      • 创建的普通文件默认没有执行权限
    • 实例:
      umask为0022          000 010 010
创建的目录文件权限      rwx r-x r-x
创建的普通文件权限      rw- r-- r--

      umask -S ( Symbolic ) 显示文件的默认权限 u=rwx,g=rx,o=rx
    • 配置:
      • 所有用户 /etc/profile
      • 单个用户 '~/.profile' 或者 ./.bash_profile

  3. lsattr

    • 作用: 查看使用chattr设置的文件属性隐藏属性 ( list file attributes on a Linux second extended file system )
    • 格式: lsattr

  4. chattr

    • 作用: 修改文件的属性，可以提高系统的稳定性 ( change file attributes on a Linux file system )
    • 格式: chattr [+-=] [选项] 文件
    • 选项与参数:
      选项: 
    + 表示添加
    - 表示去掉
    = 表示设置为,
参数:
    A 设置了A的属性后，若你有存取此文件（或目录）时，他的存取时间atime将不会被修改
    S 一般文件是非同步写入磁盘的（原理参考sync命令的说明），如果加上S这个参数时，当你进行任何文件的修改，该更改会“同步”写入磁盘中
    a 当设置a这个属性后，这个文件将只能新增数据，而不能删除也不能修改数据，只有root用户才有权限设置此属性
    c 这个属性设置后，将会自动将此文件“压缩”，读取此文件时将会自动解压缩，但是在存储的时候，会先压缩再进行存储
    d 当dump程序被执行时，设置d属性将可以使该文件（目录）不会被dump备份
    i 表示忽略，他可以让一个文件不被删除、改名、设置链接也无法写入新数据或新增数据，只有root用户才有权限设置此属性
    s 当文件设置了s属性，如果这个文件被删除，他将被完全的移除这个磁盘空间，如果误删了就无法找回
    u 与s相反，当使用u属性时，如果该文件被删除了，则数据内容其实还存在磁盘中，可以使用来救援该文件
注意: 
    1 属性设置常用的是设置a和i，而且很多属性值需要root用户才能设置
    2 xfs 文件系统仅支持 AadiS

文件特殊权限 SUID, SGID, SBIT

Set UID 的限制与功能

[root@study ~]# ls -ld /tmp ; ls -l /usr/bin/passwd

drwxrwxrwt. 14 root root 4096 Jun 16 01:27 /tmp

-rwsr-xr-x. 1 root root 27832 Jun 10 2014 /usr/bin/passwd

  • SUID权限仅仅对二进制程序( binary program )有效
  • 执行者对该程序具有x的可执行权限
  • 本权限仅在执行该程序的过程中有效( run-time )
  • 执行者将具有该程序拥有者权限( owner )的权限

Set GID 针对文件的功能

  • SGID对二进制程序有用
  • 程序执行者对于该程序需要具备x权限
  • 执行者在执行的过程中将会获得该程序群组的支持

Set GID 针对目录的功能

  • 使用者对于此目录需要同时有r与x权限
  • 使用者在此目录下的有效群组( effective group )将会变成该目录的群组
  • 用途：若使用者在此目录下具有w权限( 可新建文件 )， 则使用者所创建的新文件，该新文件的群组与此目录的群组相同

Sticky Bit 只针对目录有效

  • 使用者对于此目录有w，x权限时
  • 当使用者在该目录下创建文件或目录时，仅有自己与root才有权利删除该文件
  • 换句话说：当甲这个使用者于 A 目录是具有群组或其他人的身份，并且拥有该目录 w 的权限， 这表示“甲使用者对该目录内任何人创建的目录或文件均可进行 "删除/更名/搬移" 等动作。” 不过， 如果将 A 目录加上了 SBIT 的权限项目时， 则甲只能够针对自己创建的文件或目录进行删除/更名/移动等动作，而无法删除他人的文件。

SUID, SGID, SBIT权限设置

  • 4 为SUID
  • 2 为SGID
  • 1 为SBIT

# 6.6 文件搜索定位

  • grep
    • 作用: General Regular Expression Parser，通用规则正则表达式分析程序，是一种强调的文本搜索工具，它能使用正则表达式搜索文本，并把匹配的行打印出来。
    • 格式: grep [选项] pattern [文件名]
    • 选项:
      • -i 字母不区分大小写
      • -n 显示行号
    • 说明:
      • pattern为所要匹配的正则表达式字符串
      • 要用好grep，其实就是需要写好正则表达式
    • 实例:
      • grep ftp /etc/passwd 在/etc/passwd文件下查找包含'ftp'字符的行
      • grep 'test' d* 显示所有以d开头的文件中包含'test'的行
      • ls /bin | grep '^m' 通过管道过滤 ls /bin输出的内容，只显示以m开头的行
      • grep -i 'hello world' menu.h main.c 显示在menu.h和main.c文件中匹配hello world的行，忽略大小写
  • find
    • 作用: 最常见和最强大的查找命令，可以用它找到任何想找的文件
    • 格式: find [目录] [条件] [动作]
      • 目录: 所要搜索的目录及其所有子目录。默认为当前目录
      • 添加: 所要搜索的文件的特征
      • 动作: 对搜索结果进行特征的处理
    • 选项:
      • name 指定文件名，可以通过*模糊匹配
      • type 指定文件类型(b/c/d/p/l/f)
      • size 指定文件大小，单位可以为k/m/g，+表示大于，-表示小于
      • user 指定用户
      • group 指定用户组
      • mtime/atime/ctime 指定修改/访问/创建时间，单位为天，+表示几天前，-表示几天内
      • amin/mmin/cmin 功能同上，单位为分钟
    • 说明:
      • 如果要寻找一个普通文件，使用find会是一个不错的主意。
      • 由于find在寻找数据的时候相当的耗资源，所以没有事情不使用find

# 6.7 文件压缩解压

  • gzip
    • 作用: 压缩文件，只能是单个文件，不能是多个，也不能是目录
    • 格式: gzip file
    • 说明: 执行命令会生成file.gz，删除原来的file
    • 选项: -d 等价于gunzip
  • gunzip
    • 作用: 解压使用gzip压缩生成的文件
    • 格式: gunzip file.gz
    • 说明: 解压file.gz文件，生成file，删除原来的file.gz
  • bzip2/bunzip2
    • 说明:
      • 用法与gzip/gunzip相同，只是多了个'-k'参数，压缩或解压后保留原文件
      • 使用bzip2压缩的文件后缀为bz2，而使用gzip压缩的文件后缀为gz
  • tar
    • 说明:
      • -c 压缩
      • -x 解压
      • -z 使用gzip
      • -j 使用bzip2
      • -f 指定处理文件
      • -v 显示(压缩解压过程的)详细信息
      • -C 指定解压后存放文件的目录
    • 示例:
      • tar -zcvf 123.tar.gz 1 2 3 将目录1 2 3三个目录打包为123.tar.gz并保留原目录
      • tar -zxvf 123.tar.gz [-C ./tmp] 使用gzip将123.tar.gz解压[至./tmp目录]
